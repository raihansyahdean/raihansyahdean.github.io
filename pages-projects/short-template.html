Rube Goldberg Machine in Maya

Class Project | Spring 2024 | 3 weeks

3D Animation

Project Overview

Created a 30-second Rube Goldberg machine animation as part of Introduction to 3D Animation course. The project focused on understanding Maya's fundamentals including modeling, rigging, physics simulation, and rendering. The machine features 8 interconnected chain reactions using rigid body dynamics and keyframe animation to create a cohesive, entertaining sequence.

Duration
3 Weeks (Part-time)
Tools Used
Maya Arnold Renderer
My Role
Solo Project - All aspects

Development Process

1

Concept & Planning

  • Sketched out the chain reaction sequence on paper to visualize timing and flow
  • Decided on classic ball-rolling-down-ramp leading to dominos setup
  • Created rough blockout in Maya to test spacing and physics feasibility
  • Identified critical points where chain reactions needed careful tuning
2

Modeling & Setup

  • Modeled all objects using basic primitives and polygon modeling techniques
  • Created modular domino pieces for easy duplication and arrangement
  • Built ramps with proper angles calculated for smooth ball rolling
  • Designed simple rope constraint system for pendulum mechanics
  • Focused on clean topology for better physics simulation results
3

Physics Simulation

  • Set up rigid body dynamics for all interactive elements
  • Tuned mass, friction, and bounciness through extensive trial and error
  • Baked successful simulations to keyframes for consistency
  • Added custom constraints to pendulum for desired swing arc
  • Created hybrid approach: physics + keyframes for reliable results

Key Learning: Physics simulation in Maya requires careful balance between realism and artistic control. Sometimes "cheating" with keyframes produces better results than pure physics.

4

Lighting & Rendering

  • Implemented three-point lighting setup with Arnold lights
  • Added ambient occlusion and global illumination for realistic depth
  • Applied basic materials with varied colors to distinguish objects
  • Rendered at 1080p with 5 samples per pixel for clean results
  • Total render time: approximately 45 minutes for 30-second sequence

🚧 Main Challenge

The domino chain kept falling inconsistently due to physics glitches. Some dominos would fall too fast, others wouldn't fall at all, making the timing unpredictable and ruining the chain reaction.

✓ Solution Applied

Switched to a hybrid approach: ran physics simulation for the first 3 dominos to get natural motion, then keyframed the remaining dominos based on that timing. This gave me full control while maintaining realistic movement.

Key Learnings

  • Maya fundamentals mastered: Gained solid understanding of interface, modeling tools, animation graph editor, and render settings through hands-on practice.
  • Physics simulation workflow: Learned when to use dynamics vs keyframes, importance of baking simulations, and how object properties affect physical behavior.
  • Iteration is essential: Went through 15+ simulation attempts before achieving desired results. Each failure taught me about physics parameter tuning.
  • Planning saves time: Initial paper sketches and blockout prevented wasted effort on complex ideas that wouldn't work physically.
  • Lighting makes the difference: Simple three-point lighting elevated the entire presentation. Shadows and highlights made the 3D space feel more tangible.
