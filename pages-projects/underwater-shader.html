Stage 01

Prototyping & Foundation

Movement System Design

  • Implemented physics-based character controller using Unity's CharacterController
  • Created smooth acceleration and deceleration curves for natural movement feel
  • Added wall-sliding mechanics to prevent player frustration
  • Tuned gravity multiplier for responsive jump feel

Camera Controller

  • Developed custom third-person camera with Cinemachine
  • Implemented collision detection to prevent camera clipping
  • Added smooth camera transitions between different game states
  • Created field-of-view adjustments for speed variations

Input System Architecture

  • Utilized Unity's new Input System for cross-platform compatibility
  • Created modular input action maps for different game states
  • Implemented rebindable controls with UI feedback
  • Added controller vibration support for impact events

Stage Results

The prototyping phase successfully established the core foundation of the game's feel. Through iterative testing with 15+ playtesters, we refined the movement to achieve a responsive 8.5/10 satisfaction rating. The modular input system proved invaluable during later development, allowing seamless integration of new mechanics without refactoring. Early performance metrics showed consistent 60 FPS on target hardware with room for additional features.

Stage 02

Core Mechanics Implementation

Combat System

  • Designed combo-based melee system with 3-hit chains
  • Implemented hitbox detection using Unity's Physics.OverlapSphere
  • Created animation state machine with attack canceling windows
  • Added hit-stop effect for impactful combat feedback
  • Balanced damage values through 50+ hours of playtesting

Enemy AI Behavior

  • Built behavior tree system using custom state machine
  • Implemented pathfinding with Unity NavMesh
  • Created detection zones with line-of-sight checks
  • Added varied attack patterns for different enemy types

Stage Results

The combat system became the centerpiece of the game's identity. Player feedback indicated that combat felt "weighty and satisfying" with the hit-stop implementation proving particularly impactful. The behavior tree architecture allowed rapid iteration on enemy patterns, reducing AI development time by 40% compared to initial estimates. Combat encounters now provided consistent challenge while remaining fair and learnable.

Stage 03

Polish & Optimization

Performance Optimization

  • Implemented object pooling for frequently spawned entities (reduced garbage collection by 70%)
  • Optimized draw calls using static batching and GPU instancing
  • Created LOD system for complex meshes (30% performance improvement)
  • Profiled and optimized physics calculations in Update loops
  • Reduced texture memory usage by 45% through compression and atlasing

Visual Polish

  • Added particle effects for combat impacts and environmental interactions
  • Implemented screen shake system with customizable intensity curves
  • Created dynamic lighting for day/night cycle
  • Enhanced UI with smooth transitions and feedback animations

Stage Results

Optimization efforts yielded significant improvements across all target platforms. Frame rates stabilized at 60 FPS even during intensive combat scenarios with multiple enemies and particle effects. Memory usage was reduced by 40%, allowing the game to run smoothly on minimum spec hardware. The visual polish pass elevated the overall presentation, with testers noting the game felt "AAA quality" despite the indie development context. Load times were cut in half through asset streaming optimization.

Takeaways

Project Insights & Learnings

  • Early prototyping paid dividends: Investing 3 weeks in core feel before building features prevented months of rework and established a solid foundation that guided all subsequent decisions.
  • Modular architecture is essential: The input system and behavior tree designs proved their worth when adding new features, saving an estimated 100+ hours of refactoring work.
  • Player feedback is invaluable: Regular playtesting sessions (30+ across development) caught issues early and validated design decisions, particularly for combat timing and difficulty curves.
  • Optimization should be ongoing: Rather than leaving performance work to the end, integrating profiling into weekly workflows prevented technical debt accumulation.
  • Successfully shipped core features: All primary mechanics (movement, combat, AI) were completed and polished to production quality within the 18-month timeline.
  • Scope management was challenging: Had to cut multiplayer mode and two planned enemy types to meet deadline, reinforcing the importance of realistic scheduling and early feature prioritization.
  • Technical achievement: Achieved 60 FPS target on all platforms including older hardware, demonstrating the effectiveness of optimization strategies and LOD implementation.
