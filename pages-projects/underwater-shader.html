Underwater Shader Graphics Project

University of Southern California | October 2025 - December 2025

Project Details

This project sought to represent two different phenomena regarding how light interacts with water. To be specific, it aims to render an underwater scene featuring both water caustics and “god rays” (beams of light coming from the water’s surface). Initially, the methods for rendering the caustics and the god rays were intended to be directly linked. This is because it was believed that rendering one (in this case, the god rays) would necessarily beget the other (the caustics). However, further research indicated that these phenomena are seldom rendered in this way and are usually created with separate methods. Therefore, this project implements god rays with ray marching as a shader effect and caustics with 3-dimensional Worley noise.

Algorithm 01

Caustics Algorithm

⚠️ Problem

Need some noise generation algorithm that can simulate water caustics that changes seamlessly through time. The noise would also need to be rendered to a plane to visualize the bottom floor of the sea level.

✓ Solution

Implement an algorithm that would generate 3D Worley noise data. Each slice represents one timestep of the animation. Animating it would then only be traversing the slices on the z-axis.

3D Worley Noise

  • Using Unity's Universal Render Pipeline as the base system of the project
  • Created an algorithm that would generate 3D Worley noise
  • Using C# script to compute the values
  • Generate random points three-dimensional space to serve as the centers of the Worley cells
  • Apply nearest-neighbor algorithm to compute noise value at each point in the volume

Generating on a Plane

  • Sampling a single slice of the Worley field along the x-z plane
  • Moving along the y-axis for every slice over time
  • First iteration of the caustics shown on the video. Caustics are seen but still black and white in color.

Animating

  • By slicing the 3D Worley noise on the y-axis over time, it was able to generate seamless animation that resembles caustics
  • Added distortion so that portions of the caustics varied according to the simulated high and low tides of the ocean surface
  • The color of each pixel was subsequently determined by interpolating among three colors: white, light blue, and deep blue, to achieve the desired caustic effect.

Stage Results

This algorithm was capable to simulate realistic caustic effects that is done in real-time on a single plane. The algorithm was also able to show an animation of the caustic that is seamless and can be made to loop infinitely. In its current state, this Worley texture cannot be directly applied to the scene. The output image would need to be blended with every object's base color texture and then meticulously scaled to look even and continuous across every object.

Algorithm 02

God Rays

⚠️ Problem

Need for a way to be able to render a volume of water with varying light intensity. Some parts of the water would need to be seen as transparent while others are brighter.

✓ Solution

Implement a custom shader that utilizes raymarch to simulate god rays in a volume of water.

HLSL & Shader Graph

  • Implemented using custom HLSL shader code
  • Combine with Unity's built-in Shader Graph

Volumetric Rendering

  • Treating the water as a single homogenous volume
  • Perform raymarch in the volume of water, sampling a water density. In our case, uniform water density was assumed.
  • At each step of the primary raymarch, perform a secondary raymarch towards the light.
  • On each step of the secondary raymarch, sample a 3D Simplex noise representing light density within the volume

Beer's Law

  • Using the accumulated light density from the secondary raymarch to compute the shadow value
  • Beer's Law is the formula used to calculate final light contribution at each point in the volume
  • The accumulated final light values, taken from each step of the primary raymarch, is then used to generate the color of each pixel by interpolating between shadow color and god ray light color
  • The accumulated water density was then also applied through Beer's Law to determine the alpha value of each pixel
  • Pass the interpolated color and alpha to the fragment shader

Stage Results

This algorithm is capable to simulate realistic god rays that is done in real-time. Albeit some fine tuning of the parameters are required as the shorter the step more computations are needed. The algorithm also takes into consideration the direction of the light source. The god rays are also seen refracted inside the volume water giving a more realistic look.

Bringing it Together

Polishing & Optimization

⚠️ Problem

A good underwater scene with several objects on floor level with varying materials and textures. Caustics should be rendered on top of each object not only on the seafloor plane. God rays should also run on top of it and caustics as well as the objects can still be seen clearly.

✓ Solution

Implement a single scene and fine-tune the variables so that both algorithms can run in real time. Converted the generated Worley textures to decal material. Adjusted the volume of water and god rays algorithm parameters.

Building the Scene

  • Requires some models and textures of an underwater scene.
  • An underwater shot with caustics on the ground and on some objects.
  • A volume of water with god rays visible in it

Decal Material

  • Blend the Worley texture with every object's base color texture
  • Scale the blended texture to keep it look even and continuous across every object
  • Using Unity's decal system

Animating God Rays

  • Similar to the caustics, move over the Simplex 3D noise over time
  • Also changing the direction of the light to simulate light coming from different directions
  • Showing that some parts of the volume of water remain transparent while others exhibit lighter blue shading representing areas affected by god rays
  • God rays appear to originate from a single directional light source. But within the water, the god rays are refracted effectively and does not remain parallel

Stage Results

By building a scene and applying a decal material to our caustics, we were able to render the caustics on top of several objects. The caustics also didn't break and still look seamless when animated over time. Fine-tuning the god rays parameters also allows the scene to still be rendered particularly well with all components present. It can be seen quite well on the final render shot that the caustics are moving underneath the volume of water and on the objects in the scene while on top of them god rays are seemingly visible that are refracted in different directions but still seemingly originated from one direction.
